## 1. 🎯 결과물 체크리스트 (← 가장 중요: 무엇을 만들어야 하나)

**핵심 방향**: 모범답안(유가)을 *그대로*가 아니라 **한 단계 고도화** = 모범답안 구성 전부 **＋ 제안 ＋ Action Plan**.

| 모범답안(유가) 구성 | 우리 결과물 |
|---|---|
| SCQA 도입 | ✅ 동일 |
| 기초통계 + 검정(ANOVA·t·상관) | ✅ 동일 |
| Wilke 시각화 | ✅ 동일 |
| 피라미드 결론 | ✅ 동일 |
| 전략적 시사점(간단히) | 🔼 **제안(구체) + Action Plan(주체·시점·KPI)** 로 고도화 |

### ✅ 분석 노트북에 들어갈 것
- [ ] 데이터 정제·전처리 (재현 가능 · 주석)
- [ ] 근거 분석 4개 — 각각 `이론 1줄 → 코드 → 숫자 → 차트 → 해석 1줄`
- [ ] 기초통계(평균·중앙값·표준편차·분포) + **통계검정**(ANOVA·t검정·상관 등)
- [ ] 지도 시각화(folium) 1개 이상

### ✅ 발표 슬라이드에 들어갈 것
- [ ] 타이틀 (데이터 규모를 큰 숫자로 임팩트)
- [ ] 목차
- [ ] **SCQA** 도입 (상황→문제→질문→답)
- [ ] 근거별 분석 슬라이드 — 장마다 **Header Message(한 줄 결론)**
- [ ] **핵심 결론** (피라미드 꼭대기 한 줄)
- [ ] 🔼 **제안(Recommendations) 3개** (근거에서 도출)
- [ ] 🔼 **Action Plan** (단기·중기·장기 / 주체 / KPI)
- [ ] 섹션 인디케이터 + 일관된 디자인(색·폰트·정렬)

### ✅ 공통 / 품질
- [ ] **슬라이드 숫자 = 노트북 숫자** (QA 통과 — *모범답안도 이게 틀렸었음*)
- [ ] 차트 Wilke 통일(크기순 정렬·의미 있는 색·결론형 제목)
- [ ] *(개별)* 엘리스 퀴즈 **5명 전원** 제출
- [ ] 발표 리허설 + 파트 분담

## 8. 🎯 제안 → Action Plan (민토 "행동형" 피라미드)
분석은 "발견"에서 끝나면 절반. **"그래서 뭘 해야 한다(제안·행동)"**까지 가야 완성.
민토 피라미드 2종: **설명형**(왜 사실=분석) + **행동형**(어떻게=제안). 발표는 둘을 이어 붙임.
```
SCQA 도입 → [핵심 결론] → 진단(근거1~4) → 제안(3개) → Action Plan(단·중·장기 / 주체 / KPI)
```
> 각 Action에 **주체·시점·KPI**를 붙이면 "발견"이 아니라 **실행 가능한 제안 보고서**가 됨. (주제별 제안 대상·예시는 부록 B)
