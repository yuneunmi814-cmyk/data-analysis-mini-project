## 3. 🐙 GitHub 협업 방법 ★중요

### 폴더 구조 (사람별 파일 분리 = 충돌 차단)
```
data/                       # 공용 (M2가 정제해 올림)
  subway_time.csv           #   시간대별 마스터셋
  subway_loc.csv            #   역 좌표 (M3)
notebooks/
  01_clean_M2.ipynb         # 데이터 정제
  02_trend_M2.ipynb         # 근거1 추이
  03_line_M2.ipynb          # 근거2 호선 ANOVA
  04_peak_M3.ipynb          # 근거3 시간대 피크
  05_map_M3.ipynb           # 근거4 지도
charts/                     # M4: 정제한 차트(png) 모음
slides/                     # M4: 발표자료
docs/
  storyline_M1.md           # M1: SCQA·피라미드
  proposal_M1.md            # M1: 제안·Action Plan
README.md
```

### 작업 흐름 (브랜치 → PR)
1. 작업 전 **항상** `git pull origin main` (최신 상태로 시작)
2. 내 브랜치 만들기: `git switch -c feat/이름-내용` (예: `feat/m2-clean`)
3. **내 파일만** 수정 → `git add` → `git commit -m "[M2] 데이터 정제 완료"`
4. `git push -u origin feat/m2-clean` → GitHub에서 **Pull Request** 생성
5. **M1이 확인 후 main에 merge** (main 직접 push 금지)

### ⚠️ 노트북 충돌 황금률
- **한 `.ipynb`를 여러 명이 동시에 편집 ❌** → 각자 자기 파일만 (위 구조)
- 노트북은 출력(output) 때문에 충돌 잦음 → **커밋 전 "Restart & Clear all outputs"** 후 저장
- 커밋 메시지에 **[M1~M4]** 누가 했는지 표시

### 🔑 팀원 초대 (레포 주인이 직접)
GitHub 레포 → **Settings → Collaborators → Add people** → 팀원 3명 GitHub 아이디 추가
> ⚠️ 이건 권한 설정이라 **레포 주인이 직접** 해야 해요 (대신 못 해드림).
