# 팀 회의 안건 & 프로젝트 브리핑

- **회의**: Google Meet — https://meet.google.com/dvc-inrj-was
- **목적**: 프로젝트 현황 공유 + 역할·일정 확정 + 팀원 액션 아이템 전달
- **참석**: 2명 (나 / 팀원)

---

## 1. 프로젝트 한 장 요약
- **주제**: 서울 지하철 혼잡도 분석 (1~9호선)
- **문제**: 코로나 후 수요 재회복(2024년 2019년의 91%) → 혼잡 재심화. 혼잡이 **2호선·도심 업무지구·출퇴근 시간대**에 집중, 일부 구간은 **이태원 참사 수준 밀집**(안전 위험).
- **타겟**: 정책입안자 — **투트랙(서울교통공사 / 국회 국토교통위)**
- **결과물**: ① 분석 노트북 ② 발표 슬라이드 ③ (개별) 엘리스 퀴즈

## 2. 현재까지 진행 (이미 완료, GitHub에 있음)
✅ 문제정의 · 데이터 정제 · **분석 근거 1~4(실제 숫자)** · 차트 4종 · 제안/액션플랜 · 자료조사(출처 검증)

## 3. 공유할 핵심 발견 (회의 때 설명)
- 코로나 **–27.4%** → 2024년 **91% 회복**(재혼잡)
- **2호선 압도**(2위의 2배+), 호선 차이 통계적으로 유의(ANOVA)
- 출근 **하차 08시** / 퇴근 **승차 18시** 피크
- **상위 10% 역이 전체의 28.5%** 집중 (홍대입구·잠실·서울역·강남…)
- 교통혼잡비용 **연 81.3조원(GDP 3.37%)** → 제안의 경제 명분

## 4. 역할 분담 확인
| 역할 | 담당 | 내용 |
|---|---|---|
| 메인(분석·제작) | 나 | 데이터 분석·노트북·슬라이드·GitHub |
| 리서치·보조 | 팀원 | 자료 수집·검증, 발표 일부 (아래) |

## 5. 팀원 액션 아이템 (회의 후 진행)
- [ ] 🗺️ **역 좌표 파일 다운로드** → 폴더에 넣기
  - 표준데이터: https://www.data.go.kr/data/15013205/standard.do (1~9호선 전체)
- [ ] 📰 자료조사 **출처 검증·보강** (배경 기사·정책 사례 추가) → `docs/research/`
- [ ] 📊 발표 슬라이드 **일부 분담** (예: 배경·정책 사례 파트)
- [ ] 🎤 발표 리허설 파트 준비

## 6. 다음 일정 / 마일스톤
1. 좌표 확보 → **지도(근거4) 완성**
2. **발표 슬라이드** 제작 (SCQA→근거→제안→액션플랜)
3. 숫자 QA(슬라이드=노트북) → 리허설 → 제출

## 7. 발표 분담 (안)
- 도입(SCQA) + 핵심 결론 + 제안/액션플랜 → **나**
- 배경(코로나·안전) + 정책 사례 → **팀원**
- Q&A 데이터 백업 → 공동
