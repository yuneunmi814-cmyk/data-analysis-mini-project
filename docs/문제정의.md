# 문제 정의 — 서울 지하철 "혼잡 취약 지점"과 사고 시 혼잡 폭증

## 1. 배경
서울 지하철은 하루 수백만 명이 이용하는 핵심 교통수단. 코로나로 급감했던 수요가 **2024년 2019년의 91%까지 회복**되며 출퇴근 혼잡이 재심화되고 있다.

## 2. 문제 상황 (핵심)
평상시 혼잡도 문제지만, **진짜 위험은 이미 혼잡한 노선·역·시간대에 사고·신호장애·비상정지(급정차)가 겹칠 때** 발생한다. 운행이 막히면 승객이 승강장·계단에 한꺼번에 폭증해 **압사 위험 수준의 혼란**이 된다.

- **반복되는 출근길 사고** (한 달 새 주요 노선에서 연속 발생):
  - 2025.7.4 오전 7:50 **2호선 서울대입구역** 신호장애(선로전환기 고장) → 약 **4시간** 운행 차질, 승강장·계단 가득, 30분+ 지체 ([YTN](https://www.ytn.co.kr/_ln/0103_202507041659207090))
  - 2025.8.4 오전 7:30 **4호선 평촌역** 전기공급 장애 → **350명 환승**, 7개 열차 10분+ 지연 ([MBC](https://imnews.imbc.com/news/2025/society/article/6742218_36718.html))
  - 팀원도 사고로 인한 출근길 마비를 직접 겪음.
> 하필 **가장 혼잡한 2호선·4호선**, 그것도 **출근 피크(7시대)**에 발생 — 이미 혼잡한 노선·시간대일수록 사고 시 타격이 폭증한다.
> 📊 실제로 **5년 사고 2,837건 분석 결과: 08시 사고가 가장 많고(374건), 출퇴근 4시간에 34.4% 집중**(균등이면 21%), 사고 다발역이 혼잡 환승역과 일치한다. → 평시 혼잡 = 사고 취약성이 **데이터로 입증**됨.

> 💡 핵심 논리: **"평시 혼잡도가 높은 지점 = 사고가 겹치면 피해가 폭증하는 취약 지점."**
> 따라서 평시 혼잡 데이터는 곧 **'사고 취약성' 지도**가 된다.

평소 혼잡은 **특정 노선·역·시간대에 집중**(상위 10% 역이 전체의 **28.5%**, 2호선 압도)되어 있어, 한정된 예산·인력을 어디에 우선 투입할지가 관건이다.

## 3. 분석 질문 (가설)
1. (평시) 어느 **노선·역·시간대**가 가장 혼잡한가? → **사고가 겹치면 가장 위험한 취약 지점**은 어디인가?
2. 혼잡은 소수 지점에 **집중**되나? (선택과 집중으로 대응 가능한가)
3. 출퇴근 **피크 시간대**와 유입(업무지구)/유출(주거지) 구조는?
→ 이 취약 지점을 짚어 **① 평시 혼잡 완화 + ② 사고·이상상황 시 비상 혼잡 대응**의 우선순위를 제시.

## 4. 분석 대상 / 활용 (투트랙)
- 🚇 **서울교통공사**(운영·실행): 평시 혼잡관리 + **사고 시 비상 혼잡 대응 프로토콜**(집중역 우선), 신호설비 예방정비
- 🏛️ **국회 국토교통위원회**(입법·예산): 노후 신호설비 교체 예산, 이상상황 대응 기준 + 수요 분산 제도
> 정당색 없이 **안전·효율·예산** 논리로 작성.

## 5. 기대 효과
"평시에도 혼잡하고 **사고가 나면 더 위험해지는** 취약 지점"을 데이터로 식별 → 한정 자원의 **선택과 집중**(우선 투자 + 비상 대응) 근거 제공.

## 6. 분석 범위
- 데이터: 서울 지하철 호선별·역별·시간대별 승하차 (2015~2026, 1~9호선)
- 추이 = 전체 기간 / 혼잡·시간대·역별 = 최근월 단면
- "혼잡" = 역 승하차 인원(유동인구) 기준 (열차 내 혼잡도% 데이터는 선택 보강)
