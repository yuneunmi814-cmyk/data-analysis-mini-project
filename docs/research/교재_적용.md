# 교재 4권 → 프로젝트 적용 (정독 발췌)

> 폴더의 교재 4권을 우리 지하철 혼잡·사고 프로젝트 관점에서 정독해 **"원칙 → 적용 액션"**으로 정리. (저작권상 원문 직접 인용 없이 요약·재서술)

---

## 1. 📐 민토 피라미드 → 구조·메시지

| 원칙 | 우리 적용 액션 |
|---|---|
| **결론 먼저(Top-down)** | 결론(Governing Thought) 슬라이드를 **SCQA 직후 앞쪽**에 배치 ("투트랙으로 집중 투자하라"를 먼저 선언) |
| **MECE 그룹핑 + 귀납** | 근거2~5를 **"혼잡·사고는 4개 축(호선·시간·공간·사고)에 집중"**이라는 한 묶음으로 재배열. **근거1(연도추이)은 배경(S)으로 강등** |
| **헤드라인 = 인사이트 문장** | "호선별 분석"(라벨) ❌ → "2호선이 모든 호선을 압도한다"(문장) ⭕ — 이미 적용 중, 더 다듬기 |
| **제안 = 효과(끝산출물)로** | "OO 강화"(모호) ❌ → "첨두 혼잡 X%↓ / 사고 대응 Y분↓"(측정가능 목표) ⭕ |
| **'텅 빈 주장' 금지** | "결론/시사점" 같은 단어를 헤드라인에서 제거하고 내용을 완성 |

## 2. 🎨 Wilke 시각화 → 차트 6종 개선

**핵심 3원칙**: ①범례 → **직접 라벨** ②강조 = **accent 1색 + 나머지 회색**(Okabe-Ito 팔레트) ③막대 **0부터**·정렬(자연순서 유지)

| 차트 | 개선 액션 |
|---|---|
| 근거1 추이 | 라인 끝 직접 라벨, 코로나 회색 음영(라인 1색 유지) |
| 근거2 호선 | 값순 정렬 + **2호선만 강조색, 나머지 회색** + 막대 끝 수치 |
| 근거3 시간대 | 승/하차 라인 끝 직접 라벨(범례 삭제), 출퇴근 회색 밴드 |
| 근거4 지도 | 점 크기·색 **중복 인코딩 정리**, 색맹 안전 순차 팔레트, 겹침 투명도 |
| 근거5 사고 | 시간 순서 유지, 출퇴근만 강조색·나머지 회색, 0부터 |
| 공통 | **축·범례 글자 키우기**, 6종 색 언어 통일(2호선=항상 같은 색) |

> 팔레트(Okabe-Ito, 색맹안전): `#E69F00 #56B4E9 #009E73 #0072B2 #D55E00 #CC79A7`

## 3. 📊 Field 통계 → 분석 보강 ★실제 계산 완료

**우리 ANOVA(근거2) 강화 결과** (Field가 요구한 효과크기·사후검정·가정점검 적용):

| 항목 | 결과 | 해석 |
|---|---|---|
| ANOVA | **F(8,305)=14.8, p<.001** | 호선 간 차이 유의 |
| **효과크기 η²** | **0.279** | **대(大) 효과** — 호선이 이용객 변동의 **28%** 설명 |
| ω² | 0.260 | (덜 부풀려진 추정) 역시 대 효과 |
| 등분산(Levene) | W=8.90, **p<.001 → 위반** | 호선별 분산 다름 |
| → 대안 **Welch's F** | **≈10.7 (유의)** | 등분산 위반 보정해도 결론 동일 |
| 사후검정(Tukey) | **2호선 > 6개 호선 유의** | "2호선 압도" 통계로 확증 |

**보고 문구(APA)**: *"호선에 따라 역당 이용객에 유의한 차이가 있었다, F(8,305)=14.8, p<.001, η²=.28(대 효과). 등분산이 위배되어(Levene p<.001) Welch's F로 재확인했고(F≈10.7, p<.001), Tukey 사후검정 결과 2호선이 6개 호선보다 유의하게 많았다."*

> ⚠️ **주의**: 표본이 커서 "p<.001"은 거의 자동 → **효과크기(η²=.28 대효과)**를 함께 말해야 설득력. "p<.001이라 매우 유의" 식 표현 자제.

## 4. 🚀 ISLP → 고도화 기법 (한 단계 위)

| 순위 | 기법 | 우리 적용 메시지 | 난이도 |
|---|---|---|---|
| 🥇 | **K-means 군집** | 역을 **업무지구형/주거형/혼합형**으로 자동 분류 → 유형별 맞춤 정책 제안 | 낮음(2일 OK) |
| 🥈 | **포아송 회귀** | 사고·혼잡 **건수**를 시간대로 정량화("출근=비피크 ○배") | 중간 |
| 🥉 | PCA | 시간대 변수 압축 → 군집 2D 시각화 보조 | 보조용 |

- K-means 주의: **표준화 필수**, `n_init` 여러 번, K=3~5 비교
- 포아송 주의: 과산포 가능 → p값 과신 금지
- ❌ 욕심 금지: SVM·딥러닝·트리는 2일 발표에 부적합(해석성 우선)

## 5. ✅ 우선 적용 액션 (효과·난이도순)
1. **(즉시·완료) 근거2 통계 강화** — η²=.28 대효과 + Welch + Tukey → 노트북·장표에 반영
2. **(빠름) 차트 6종 Wilke 개선** — 직접 라벨·accent+회색·글자 키우기
3. **(빠름) 헤드라인·결론 먼저 재배열** — 민토식 메시지 정리
4. **(중간·선택) K-means 역 군집(근거6)** — "역 유형" 새 인사이트 + 유형별 제안
5. **(선택) 포아송 회귀** — 사고 건수 정량화
